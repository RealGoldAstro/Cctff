Custom Chat Text Formatting Framework (CCTFF)

CCTFF is a configuration-driven Roblox chat tags system that centralizes all tag rules, prefix formats, leaderboard positions, system notifications, and color themes into one CustomChatTagsConfig table. Update any tag or style once and see changes reflected across all chat channels and bubbles instantly.

Whether you need role tags, dynamic leaderboard ranks, or server-wide system messages, CCTFF handles everything with persistent DataStore support, automatic syncing, and zero client-side guesswork.

Framework Structure

CCTFF is organized into three core script locations in your Roblox game:

  • game.ReplicatedStorage
    • CustomChatTagsAPI ‚Äì Server-side tag lifecycle (give/remove/equip/unequip), validation, and events.
    • CustomChatTagsClientAPI ‚Äì Safe client proxy that mirrors server methods via RemoteFunction.
    • CustomChatTagsConfig ‚Äì Central configuration for all tags, leaderboard tags, system tags, styles, and display rules.
  • game.ServerScriptService
    • CustomChatTagsDataStore.server ‚Äì DataStore persistence, caching, auto-save, and retry logic.
    • CustomChatTagsServer.server ‚Äì Network event handlers, tag update broadcasts, and system message delivery.
  • game.StarterPlayer.StarterPlayerScripts
    • CustomChatTagsClient.client ‚Äì Live chat rendering, bubble customization, and RichText formatting for messages.

Three Core Features

CCTFF supports three distinct categories of chat enhancements, each with unique APIs and styling options:

1. Player Tags

Player tags are role-based or achievement-based identifiers displayed before usernames. Tags can be temporary (auto-expire) or permanent, and players can equip up to 3 tags at once (configurable).

Example Configuration


CustomChatTagsConfig.Tags = {
  Developer = {
    Enabled = true,
    Priority = 1,
    TagText = "[DEVELOPER]",
    TagColor = Color3.fromRGB(100, 255, 100),
    UsernameColor = Color3.fromRGB(255, 255, 255),
    TextColor = Color3.fromRGB(100, 255, 100),
    ChatBubble = {
      BackgroundColor = Color3.fromRGB(0, 0, 0),
      StrokeColor = Color3.fromRGB(100, 255, 100),
      CornerRadius = 12,
    },
  },
  Owner = {
    Enabled = true,
    Priority = 3,
    TagText = "[Owner]",
    TagColor = Color3.fromRGB(255, 0, 0),
    UsernameColor = Color3.fromRGB(231, 19, 19),
    TextColor = Color3.fromRGB(255, 0, 0),
  },
}

Chat Display Example

[DEVELOPER] PlayerName: Hey everyone, check out the new update!

Server Usage


local TagAPI = require(game.ReplicatedStorage.CustomChatTagsAPI)

game.Players.PlayerAdded:Connect(function(player)
  TagAPI:Give(player, "Developer")
  TagAPI:Equip(player, "Developer")
end)

2. Leaderboard Tags

Leaderboard tags display dynamic rank positions next to player names. The tag text updates in real-time based on player stats (e.g., top coin holders). Use %s in the tag template to inject rank text.

Example Configuration


CustomChatTagsConfig.LeaderboardTags = {
  TopCoins = {
    Enabled = true,
    Priority = 5,
    TagText = "[üí∞ Coins #%s]",
    TagColor = Color3.fromRGB(255, 215, 0),
    UsernameColor = Color3.fromRGB(255, 215, 0),
    TextColor = Color3.fromRGB(255, 215, 0),
    ChatBubble = {
      StrokeColor = Color3.fromRGB(255, 215, 0),
      CornerRadius = 12,
    },
  },
}

Chat Display Example

[üí∞ Coins #1] TopPlayer: I'm first place!

Server Usage


local TagAPI = require(game.ReplicatedStorage.CustomChatTagsAPI)

-- Give leaderboard tag with dynamic rank text
TagAPI:GiveLeaderboardTag(player, "TopCoins", "1")

-- Update rank when leaderboard changes
TagAPI:UpdateLeaderboardTag(player, "TopCoins", "2")

3. System Messages

System messages are server-wide notifications displayed in chat without a player sender. Perfect for announcements, alerts, errors, or event notifications. Players can toggle visibility per message type.

Server Usage


local TagAPI = require(game.ReplicatedStorage.CustomChatTagsAPI)

-- Send system message to all players
local success = TagAPI:SendSystemMessage(
  "ServerRestart",  -- Unique message name
  "System",         -- System tag to use
  "Server restart in 5 minutes. Save your progress!"
)

Client Usage (Toggle Messages)


local ClientAPI = require(game.ReplicatedStorage.CustomChatTagsClientAPI)

-- Disable specific system messages
ClientAPI:UpdateClientSettings({
  ServerRestart = false,  -- Hide restart warnings
  EventStart = true,      -- Show event notifications
})

Key Benefits

  • Zero Hardcoding: All tags, colors, and priorities defined in one config table.
  • Automatic Sync: Tag changes on the server instantly update for all clients.
  • DataStore Persistence: Player tags, equipped slots, and preferences saved automatically.
  • Rich Styling: Control text color, bold/italic/underline, stroke thickness, and chat bubble appearance per tag.
  • Dynamic Leaderboard: Update player ranks in real-time without re-giving tags.
  • Flexible System Messages: Per-player visibility controls for notifications.

Quick Start

Getting started with CCTFF takes just three steps:

  1. Place all CCTFF modules in the correct Roblox locations (ReplicatedStorage, ServerScriptService, StarterPlayerScripts).
  2. Edit CustomChatTagsConfig to define your tags, leaderboard tags, and system tags.
  3. Call TagAPI:Give(player, "TagName") and TagAPI:Equip(player, "TagName") from your server scripts.

All rendering, formatting, and client updates happen automatically. For detailed API documentation, explore the sidebar navigation for each method's parameters, return values, and usage examples.
