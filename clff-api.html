CLFF API Reference

All CLFF API methods live on de>_G.CLFF and must be called from server scripts only. The API provides secure stat updates with built-in validation, type checking, and rate limiting.

API

de>CLFF:Update

Update a stat value with validation and type checking.

de>CLFF:Get

Retrieve the current value of a stat.

de>CLFF:Increment

Add or subtract from a numeric stat.

de>CLFF:Set

Alias for Update() for clarity in code.

de>CLFF:SetDebugMode

Enable or disable debug warnings for troubleshooting.

de>CLFF:SetSortingStat

Change which stat determines player order.

CLFF:Update

The de>CLFF:Update method sets a stat to a specific value with full server-side validation. This is the primary method for updating stats.

Overview

Execution Context
de>Server
Script Type
Server Script
Returns
de>boolean
Side Effects
Updates leaderstats value, broadcasts to clients

Syntax

de>local success = _G.CLFF:Update(player: Player, statName: string, newValue: number)

Parameter Type Description
de>player de>Player The player instance whose stat should be updated
de>statName de>string The name of the stat (e.g., "Coins", "Kills")
de>newValue de>number The new value to set (validated against stat config)

Return Values

Return Type Description
de>success de>boolean True if update succeeded, false if validation failed

Basic Usage

Update a player's coins with error checking.

Lua 
de class="language-lua">local success = _G.CLFF:Update(player, "Coins", 10000)

if success then
    print("Coins updated successfully")
else
    warn("Failed to update coins")
end

DataStore Integration

Load saved player data and update all stats.

Lua 
de class="language-lua">local DataStoreService = game:GetService("DataStoreService")
local PlayerData = DataStoreService:GetDataStore("PlayerData")

game.Players.PlayerAdded:Connect(function(player)
    local success, data = pcall(function()
        return PlayerData:GetAsync(player.UserId)
    end)
    
    if success and data then
        _G.CLFF:Update(player, "Coins", data.Coins or 0)
        _G.CLFF:Update(player, "Kills", data.Kills or 0)
        _G.CLFF:Update(player, "Deaths", data.Deaths or 0)
        _G.CLFF:Update(player, "Level", data.Level or 1)
        _G.CLFF:Update(player, "Wins", data.Wins or 0)
    end
end)

Purchase System

Handle item purchases by deducting coins.

Lua 
de class="language-lua">local function handlePurchase(player, itemName, itemPrice)
    local currentCoins = _G.CLFF:Get(player, "Coins")
    
    if currentCoins >= itemPrice then
        local newCoins = currentCoins - itemPrice
        local success = _G.CLFF:Update(player, "Coins", newCoins)
        
        if success then
            -- Give the item to player
            giveItem(player, itemName)
            return true, "Purchase successful!"
        else
            return false, "Failed to update coins"
        end
    else
        return false, "Not enough coins"
    end
end

-- Usage
local success, message = handlePurchase(player, "Sword", 500)
print(message)

Quest Reward System

Award multiple stats when a quest is completed.

Lua 
de class="language-lua">local function completeQuest(player, questData)
    -- Award coins
    local currentCoins = _G.CLFF:Get(player, "Coins")
    _G.CLFF:Update(player, "Coins", currentCoins + questData.CoinReward)
    
    -- Award XP and level up if needed
    local currentLevel = _G.CLFF:Get(player, "Level")
    local newLevel = currentLevel + questData.LevelReward
    _G.CLFF:Update(player, "Level", newLevel)
    
    -- Track quest completion
    local currentWins = _G.CLFF:Get(player, "Wins")
    _G.CLFF:Update(player, "Wins", currentWins + 1)
    
    print(player.Name .. " completed quest: " .. questData.Name)
end

-- Example quest
completeQuest(player, {
    Name = "Defeat 10 Zombies",
    CoinReward = 1000,
    LevelReward = 2
})

CLFF:Get

The de>CLFF:Get method retrieves the current value of a player's stat. Returns de>nil if the stat or player doesn't exist.

Syntax

de>local value = _G.CLFF:Get(player: Player, statName: string)

Basic Usage

Get a player's current coin balance.

Lua 
de class="language-lua">local coins = _G.CLFF:Get(player, "Coins")

if coins then
    print(player.Name .. " has " .. coins .. " coins")
else
    warn("Could not retrieve coins for " .. player.Name)
end

Permission Check

Check if player has enough level to access an area.

Lua 
de class="language-lua">local REQUIRED_LEVEL = 10

local function canEnterArea(player)
    local playerLevel = _G.CLFF:Get(player, "Level")
    
    if playerLevel and playerLevel >= REQUIRED_LEVEL then
        return true
    else
        return false, "You need level " .. REQUIRED_LEVEL .. " to enter"
    end
end

-- Usage
local canEnter, errorMsg = canEnterArea(player)
if not canEnter then
    print(errorMsg)
end

Leaderboard Generation

Get all players' stats to create a custom leaderboard.

Lua 
de class="language-lua">local function getTopPlayers(statName, count)
    local playerStats = {}
    
    for _, player in ipairs(game.Players:GetPlayers()) do
        local value = _G.CLFF:Get(player, statName)
        if value then
            table.insert(playerStats, {
                Name = player.Name,
                Value = value
            })
        end
    end
    
    -- Sort by value descending
    table.sort(playerStats, function(a, b)
        return a.Value > b.Value
    end)
    
    -- Return top N players
    local topPlayers = {}
    for i = 1, math.min(count, #playerStats) do
        table.insert(topPlayers, playerStats[i])
    end
    
    return topPlayers
end

-- Get top 5 richest players
local topRich = getTopPlayers("Coins", 5)
for i, data in ipairs(topRich) do
    print(i .. ". " .. data.Name .. " - " .. data.Value .. " coins")
end

CLFF:Increment

The de>CLFF:Increment method adds or subtracts from a numeric stat. Perfect for rewards, penalties, or continuous stat updates. Use negative values to decrement.

Syntax

de>local success = _G.CLFF:Increment(player: Player, statName: string, amount: number)

Basic Reward

Give players coins for completing an action.

Lua 
de class="language-lua">-- Player defeats an enemy
_G.CLFF:Increment(player, "Coins", 100)
_G.CLFF:Increment(player, "Kills", 1)

print(player.Name .. " earned 100 coins and 1 kill!")

Combat System

Track kills and deaths in PvP combat.

Lua 
de class="language-lua">local function onPlayerKilled(killer, victim)
    -- Update killer stats
    _G.CLFF:Increment(killer, "Kills", 1)
    _G.CLFF:Increment(killer, "Coins", 25) -- Bonus coins
    
    -- Update victim stats
    _G.CLFF:Increment(victim, "Deaths", 1)
    
    -- Notify players
    print(killer.Name .. " eliminated " .. victim.Name)
end

-- Connect to your death event
game.Players.PlayerAdded:Connect(function(player)
    player.CharacterAdded:Connect(function(character)
        local humanoid = character:WaitForChild("Humanoid")
        humanoid.Died:Connect(function()
            local killerPlayer = getKiller(player) -- Your kill detection
            if killerPlayer then
                onPlayerKilled(killerPlayer, player)
            end
        end)
    end)
end)

Penalty System

Deduct coins for penalties using negative increments.

Lua 
de class="language-lua">local function applyDeathPenalty(player)
    local currentCoins = _G.CLFF:Get(player, "Coins")
    local penalty = math.floor(currentCoins * 0.1) -- 10% penalty
    
    -- Deduct coins (negative increment)
    _G.CLFF:Increment(player, "Coins", -penalty)
    
    print(player.Name .. " lost " .. penalty .. " coins")
end

-- Call on player death
humanoid.Died:Connect(function()
    applyDeathPenalty(player)
end)

Daily Login Bonus

Reward players for logging in each day.

Lua 
de class="language-lua">local function grantDailyBonus(player, dayStreak)
    local baseReward = 100
    local streakBonus = dayStreak * 50
    local totalReward = baseReward + streakBonus
    
    -- Grant bonus coins
    _G.CLFF:Increment(player, "Coins", totalReward)
    
    -- Notify player
    print(player.Name .. " claimed " .. totalReward .. " coins!")
    print("Day " .. dayStreak .. " streak bonus!")
end

-- Example: Player logs in on day 5 of streak
grantDailyBonus(player, 5) -- Grants 100 + (5 * 50) = 350 coins

CLFF:Set

The de>CLFF:Set method is an alias for de>CLFF:Update. Use it when you want clearer code that explicitly shows you're setting a value.

Syntax

de>local success = _G.CLFF:Set(player: Player, statName: string, value: number)

Basic Usage

Set a player's level to a specific value.

Lua 
de class="language-lua">-- Reset player to level 1
_G.CLFF:Set(player, "Level", 1)

-- Set to max level
_G.CLFF:Set(player, "Level", 100)

Admin Commands

Allow admins to set player stats directly.

Lua 
de class="language-lua">local function adminSetStat(admin, targetPlayer, statName, value)
    if not isAdmin(admin) then
        return false, "You don't have permission"
    end
    
    local success = _G.CLFF:Set(targetPlayer, statName, value)
    
    if success then
        return true, "Set " .. targetPlayer.Name .. "'s " .. statName .. " to " .. value
    else
        return false, "Failed to set stat"
    end
end

-- Example: Admin sets target's coins to 10000
local success, msg = adminSetStat(adminPlayer, targetPlayer, "Coins", 10000)
print(msg)

Reset Player Stats

Reset all stats to default values.

Lua 
de class="language-lua">local function resetPlayerStats(player)
    _G.CLFF:Set(player, "Coins", 0)
    _G.CLFF:Set(player, "Kills", 0)
    _G.CLFF:Set(player, "Deaths", 0)
    _G.CLFF:Set(player, "Level", 1)
    _G.CLFF:Set(player, "Wins", 0)
    
    print(player.Name .. "'s stats have been reset")
end

-- Usage
resetPlayerStats(player)

CLFF:SetDebugMode

The de>CLFF:SetDebugMode method enables or disables debug warnings. Useful for troubleshooting during development.

Syntax

de>_G.CLFF:SetDebugMode(enabled: boolean)

Enable Debug Mode

Turn on debug warnings to see detailed logs.

Lua 
de class="language-lua">-- Enable debug mode
_G.CLFF:SetDebugMode(true)

-- Now all operations will print debug info
_G.CLFF:Update(player, "Coins", 1000)
-- Output: [CLFF Server] Stat updated: PlayerName - Coins = 1000

Disable Debug Mode

Turn off debug warnings for production.

Lua 
de class="language-lua">-- Disable debug mode before publishing game
_G.CLFF:SetDebugMode(false)

Development Toggle

Automatically enable debug mode in Studio only.

Lua 
de class="language-lua">local RunService = game:GetService("RunService")

-- Enable debug mode only in Studio
if RunService:IsStudio() then
    _G.CLFF:SetDebugMode(true)
    warn("[CLFF] Debug mode enabled (Studio)")
else
    _G.CLFF:SetDebugMode(false)
end

CLFF:SetSortingStat

The de>CLFF:SetSortingStat method changes which stat determines player order on the leaderboard. Updates take effect immediately for all players.

Syntax

de>local success = _G.CLFF:SetSortingStat(statName: string)

Basic Usage

Change sorting from Coins to Kills.

Lua 
de class="language-lua">-- Sort by kills instead
local success = _G.CLFF:SetSortingStat("Kills")

if success then
    print("Leaderboard now sorted by Kills")
else
    warn("Invalid stat name")
end

Event-Based Sorting

Change sorting based on game events.

Lua 
de class="language-lua">local function startPvPEvent()
    -- During PvP events, sort by kills
    _G.CLFF:SetSortingStat("Kills")
    print("PvP Event Started! Leaderboard sorted by Kills")
end

local function endPvPEvent()
    -- After event, sort by coins again
    _G.CLFF:SetSortingStat("Coins")
    print("PvP Event Ended! Leaderboard sorted by Coins")
end

-- Example: 10 minute PvP event
startPvPEvent()
task.wait(600) -- 10 minutes
endPvPEvent()

Seasonal Leaderboard

Rotate sorting stat based on season or day of week.

Lua 
de class="language-lua">local function updateSeasonalSort()
    local dayOfWeek = os.date("*t").wday
    
    if dayOfWeek == 1 or dayOfWeek == 7 then
        -- Weekend: Sort by wins
        _G.CLFF:SetSortingStat("Wins")
        print("Weekend Mode: Sorted by Wins")
    else
        -- Weekday: Sort by coins
        _G.CLFF:SetSortingStat("Coins")
        print("Weekday Mode: Sorted by Coins")
    end
end

-- Update every hour
while true do
    updateSeasonalSort()
    task.wait(3600) -- 1 hour
end

Security & Validation

CLFF includes multiple layers of security to prevent exploits and ensure data integrity.

Built-In Security Features

  • Server-Only Execution: API can only be called from server scripts
  • Type Validation: Checks value matches stat's ValueType (IntValue, NumberValue, etc.)
  • Sanity Checks: Rejects NaN, Infinity, and out-of-bounds values (-1e15 to 1e15)
  • Stat Name Validation: Only allows stats defined in CLFFConfig
  • Player Validation: Ensures player exists and has leaderstats
  • Rate Limiting: Throttles updates to prevent spam (configurable via UpdateThrottle)
